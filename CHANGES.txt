# Изменения в проекте

## 1. Умное сканирование с 10 попытками (main.py)
### Проблема
- Если в тройке пробирок одна отсутствует по матрице, а вторая не считалась - не было 10 попыток
- Сканер мог вернуть только 2 баркода из 3 и система не повторяла попытку

### Решение
- Функция `scan_three_positions()` теперь:
  - Принимает информацию о матрице (row, cols_range, matrix)
  - Знает сколько пробирок ожидается в каждой позиции
  - Делает до 10 попыток, пока не найдёт ВСЕ ожидаемые пробирки
  - Сохраняет лучший результат из всех попыток
  - Логирует прогресс: "3/3 пробирок", "2/3 пробирок" и т.д.

### Пример
```
Матрица: [1, 0, 1] - ожидаем пробирки в позициях 0 и 2
Попытка 1: ['ABC123', 'NoRead', 'NoRead'] - нашли 1/2, повтор
Попытка 2: ['ABC123', 'NoRead', 'DEF456'] - нашли 2/2, успех!
```

## 3. Перемещение робота при паузе (robot_controller.py, main.py, server.py)
### Проблема
- При нажатии "Пауза" робот оставался на месте

### Решение
- Веб-интерфейс устанавливает флаг `pause_requested`
- При следующем вызове `check_pause()` программа:
  1. Видит флаг `pause_requested`
  2. Перемещает робота в PAUSE_POSITION
  3. Устанавливает `paused = True`
  4. Ждёт снятия паузы
- Передаём `cobot` и `pause_position` в каждый `check_pause()`

## 4. Удаление robot_state.json при запуске (server.py)
- При старте `server.py` удаляется `robot_state.json`
- Обеспечивает чистое состояние системы

## Файлы изменены:
- main.py - умное сканирование, передача pause_position в check_pause
- robot_controller.py - флаг pause_requested, перемещение при паузе
- Index.html - кнопка возобновления
- style.css - стили для кнопки возобновления
- server.py - set_pause_requested вместо set_paused, удаление state при старте
